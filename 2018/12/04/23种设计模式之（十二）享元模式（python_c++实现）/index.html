#享元模式（Flyweight）

##概念

享元模式主要用于减少创建对象的数量，以减少内存占用和提高性能。这种类型的设计模式属于结构型模式，它提供了减少对象数量从而改善应用所需的对象结构的方式，尝试重用现有的同类对象，如果未找到匹配的对象，则创建新对象。

GoF对享元模式的定义是：运用共享技术有效地支持大量细粒度的对象。

##应用场景

（1）、系统有大量相似对象。

（ 2）、需要缓冲池的场景。
